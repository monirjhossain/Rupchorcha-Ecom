2. Database Structure (Core Tables)
-----------------------------------

**Users**
- id (PK)
- name
- email (unique)
- password
- phone
- address
- role (customer, admin, seller)
- created_at, updated_at

**Products**
- id (PK)
- name
- description
- price
- stock
- category_id (FK)
- brand_id (FK)
- image_url
- status (active/inactive)
- created_at, updated_at

**Categories**
- id (PK)
- name
- parent_id (nullable, FK)
- created_at, updated_at

**Brands**
- id (PK)
- name
- created_at, updated_at

**Orders**
- id (PK)
- user_id (FK)
- total_amount
- status (pending, paid, shipped, delivered, cancelled)
- payment_method
- shipping_address
- created_at, updated_at

**Order_Items**
- id (PK)
- order_id (FK)
- product_id (FK)
- quantity
- price
- created_at, updated_at

**Carts**
- id (PK)
- user_id (FK)
- created_at, updated_at

**Cart_Items**
- id (PK)
- cart_id (FK)
- product_id (FK)
- quantity
- created_at, updated_at

**Reviews**
- id (PK)
- user_id (FK)
- product_id (FK)
- rating
- comment
- created_at, updated_at

**Payments**
- id (PK)
- order_id (FK)
- amount
- payment_method
- payment_status
- transaction_id
- created_at, updated_at

**Wishlists**
- id (PK)
- user_id (FK)
- product_id (FK)
- created_at

3. Core Features
----------------

**Customer Features**
- User registration, login, password reset
- Product browsing, search, filter, sort
- Product details, reviews, ratings
- Add to cart, update cart, remove from cart
- Wishlist management
- Checkout, order placement
- Payment gateway integration (SSLCommerz, Stripe, etc.)
- Order tracking, order history
- Profile management

**Admin Features**
- Dashboard (sales, orders, users, products stats)
- Product CRUD (add, edit, delete, manage stock)
- Category & brand management
- Order management (view, update status)
- User management (block/unblock, roles)
- Review moderation
- Reports (sales, inventory, users)

**Seller Features (if multi-vendor)**
- Product management
- Order management
- Earnings report

**General Features**
- Responsive UI (mobile-friendly)
- Secure authentication (JWT, Sanctum)
- RESTful API (Laravel API resources)
- Email notifications (order, registration, password reset)
- SEO friendly URLs
- Multi-language support (optional)
- Role-based access control

4. Development Workflow
-----------------------

1. **Backend Setup**
   - Laravel installation
   - Database migration & seeding
   - API development (RESTful endpoints)
   - Authentication (Laravel Sanctum/Passport)
   - Unit & feature testing

2. **Frontend Setup**
   - React app setup (Vite/CRA/Next.js)
   - State management (Redux/Context)
   - API integration (Axios/Fetch)
   - Routing (React Router)
   - UI library (MUI, Ant Design, Tailwind CSS)
   - Responsive design

3. **Deployment**
   - Backend: Deploy on VPS/Cloud (DigitalOcean, AWS, etc.)
   - Frontend: Deploy on Vercel/Netlify/Shared hosting
   - Environment variables management
   - SSL, domain setup

5. Security & Best Practices
----------------------------

- Use HTTPS everywhere
- Validate & sanitize all inputs
- Use Laravel policies/gates for authorization
- Hash passwords (bcrypt/argon2)
- Rate limiting for APIs
- CSRF/XSS protection
- Regular backups

6. Future Enhancements
